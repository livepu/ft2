# 回测框架开发计划

## 1. 项目概述

### 1.1 项目定位
这是一个基于Python的量化交易回测框架，设计上兼容掘金API，支持多频率数据和多种策略类型。框架采用模块化设计，核心组件包括上下文管理、账户管理、回测引擎和结果分析器。

### 1.2 核心组件

| 组件 | 文件 | 功能 | 状态 |
|-----|------|------|------|
| **Context** | storage.py | 管理全局状态、数据缓存和订阅信息 | ✅ 已完成 |
| **AccountManager** | account.py | 处理订单、管理持仓和记录交易 | ✅ 已完成 |
| **Engine** | engine.py | 加载历史数据并按时间轴运行策略 | ✅ 已完成 |
| **AccountAnalyzer** | analyzer.py | 分析回测结果，生成各种指标和HTML报告 | ✅ 已完成 |
| **Notebook** | notebook/ | 交互式分析笔记本 | ✅ 已完成 |

### 1.3 工作流程

```
1. 初始化策略
   ↓
2. 订阅数据（context.subscribe）
   ↓
3. 加载历史数据（engine.add_data）
   ↓
4. 运行回测（engine.run）
   ├─ 按时间轴推进
   ├─ 调用策略的 on_bar 方法
   └─ 记录账户快照
   ↓
5. 分析结果（AccountAnalyzer）
   ↓
6. 生成HTML报告
```

## 2. 已完成功能

- 事件驱动架构，按时间轴推进

- 自动缓存管理

**核心特性：**
- Context 全局状态管理

### 2.4 分析器 (analyzer.py)

**核心特性：**

**核心特性：**
- 交互式分析笔记本

## 3. 架构设计原则

### 3.1 核心原则

```
┌─────────────────────────────────────────────────────────┐
│                    ft2 架构                              │
│                                                         │
│  core/              ← 回测核心，保持精简                  │
│  ├── engine.py          (回测引擎，~80行)                │
│  ├── account.py         (账户管理)                       │
│  ├── storage.py         (Context + 缓存)                 │
│  └── analyzer.py        (分析器)                         │
│                                                         │
│  ml/                ← ML模块，独立可选                    │
│  ├── features.py        (特征工程)                       │
│  ├── model.py           (模型封装)                       │
│  └── trainer.py         (训练流程)                       │
│                                                         │
│  策略层调用：model.predict() → 回测引擎执行               │
└─────────────────────────────────────────────────────────┘
```

### 3.2 设计理念

1. **回测引擎不感知 ML** - 只负责执行策略逻辑
2. **ML 模块不感知回测** - 只负责特征和模型
3. **策略层做粘合** - 调用 ML 模块获取信号，调用回测引擎下单

### 4.3 模型封装 (model.py)

**接口设计：**

```python
class ModelWrapper(ABC):
    """模型封装抽象接口"""
    
    @abstractmethod
    def fit(self, X, y) -> 'ModelWrapper':
        """训练模型"""
        pass
    
    @abstractmethod
    def predict(self, X) -> np.ndarray:
        """预测"""
        pass
    
    @abstractmethod
    def predict_proba(self, X) -> np.ndarray:
        """预测概率"""
        pass
    
    @abstractmethod
    def save(self, path: str) -> None:
        """保存模型"""
        pass
    
    @abstractmethod
    def load(self, path: str) -> 'ModelWrapper':
        """加载模型"""
        pass


class SklearnWrapper(ModelWrapper):
    """sklearn 模型适配器"""
    
    def __init__(self, model):
        """
        Args:
            model: sklearn 兼容模型（如 RandomForest, XGBoost, LightGBM）
        """
        self.model = model
    
    def fit(self, X, y):
        self.model.fit(X, y)
        return self
    
    def predict(self, X):
        return self.model.predict(X)
    
    def predict_proba(self, X):
        return self.model.predict_proba(X)
    
    def save(self, path):
        import joblib
        joblib.dump(self.model, path)
    
    def load(self, path):
        import joblib
        self.model = joblib.load(path)
        return self
```

    def on_bar(self, context, bars):

## 7. 注意事项

### 7.1 设计原则

1. **保持 core 精简** - 回测引擎不引入 ML 依赖
2. **ML 模块可选** - 不使用 ML 功能时无需安装相关依赖
3. **向后兼容** - 现有策略代码无需修改

### 7.2 性能考虑

- 特征计算使用向量化操作
- 模型推理避免重复计算
- 缓存已计算的特征

## 8. 总结

本开发计划聚焦于：

1. **回测框架已完成** - core 模块功能完整，分析器支持丰富的指标
2. **ML 模块独立** - 作为可选扩展，不影响回测核心
3. **松耦合设计** - 回测引擎和 ML 模块可独立演进

通过这种设计，框架保持精简，同时具备 AI 策略扩展能力。
