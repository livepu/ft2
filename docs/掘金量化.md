## 1. 回测运行函数 - `run()`

`run()` 是启动策略的主入口，包含多个 `backtest_` 前缀的回测参数。

### 函数签名

```python
run(
    strategy_id='',           # 策略ID
    filename='',              # 策略文件名
    mode=MODE_UNKNOWN,        # 模式：MODE_BACKTEST / MODE_LIVE
    token='',                 # 用户token

    # 回测参数
    backtest_start_time='',        # 回测开始时间，如 '2010-01-01 08:00:00'
    backtest_end_time='',          # 回测结束时间，如 '2023-12-31 15:30:00'
    backtest_initial_cash=1000000, # 回测初始资金，默认100万
    backtest_transaction_ratio=1,  # 成交比例，默认100%
    backtest_commission_ratio=0,  # 佣金比例，默认0（股票为0.0003）
    backtest_commission_unit=0,   # 单位佣金(元)，默认0
    backtest_slippage_ratio=0,    # 滑点比例，默认0
    backtest_marginfloat_ratio1=0.2,  # 保证金上浮比例1(距到期日>2天)
    backtest_marginfloat_ratio2=0.4,  # 保证金上浮比例2(距到期日<=2天)
    backtest_adjust=ADJUST_NONE,  # 复权模式：ADJUST_NONE / ADJUST_PREV / ADJUST_POST
    backtest_check_cache=1,       # 是否检查缓存
    backtest_match_mode=0,        # 撮合模式：0=开盘价/1=收盘价
    backtest_intraday=0,          # 是否日内模式
)
```

### 回测参数详解

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `backtest_start_time` | str | '' | 回测开始时间，格式 'YYYY-MM-DD HH:MM:SS' |
| `backtest_end_time` | str | '' | 回测结束时间，格式 'YYYY-MM-DD HH:MM:SS' |
| `backtest_initial_cash` | float | 1000000 | 回测初始资金 |
| `backtest_transaction_ratio` | float | 1 | 成交比例，0-1之间 |
| `backtest_commission_ratio` | float | 0 | 佣金比例（如0.0003表示万三） |
| `backtest_commission_unit` | float | 0 | 最低佣金（元） |
| `backtest_slippage_ratio` | float | 0 | 滑点比例 |
| `backtest_adjust` | int | ADJUST_NONE | 复权模式 |
| `backtest_match_mode` | int | 0 | 撮合模式：0=以开盘价/1=以收盘价 |
| `backtest_intraday` | int | 0 | 是否日内模式（期货） |

### 复权常量

| 常量 | 说明 |
|------|------|
| `ADJUST_NONE` | 不复权 |
| `ADJUST_PREV` | 前复权 |
| `ADJUST_POST` | 后复权 |

### 模式常量

| 常量 | 说明 |
|------|------|
| `MODE_BACKTEST` | 回测模式 |
| `MODE_LIVE` | 实时模式 |

### 撮合模式说明

- `backtest_match_mode=0`：以当前bar的开盘价撮合（更保守）
- `backtest_match_mode=1`：以当前bar的收盘价撮合（更激进）

## 2. 区间收益计算 - 基准日与回测起点

### 计算公式

掘金量化的区间收益计算采用以下公式：

```
收益率 = pnl / cum_inout
     = (期末资产 - 基准资产) / 基准资产
```

其中：
- `pnl`：累计盈亏
- `cum_inout`：累计投入本金

### 基准日确定规则

| 场景 | 基准日 | 计算区间起点 |
|------|--------|--------------|
| 回测区间从数据第一天开始 | 回测区间**第一天**（起点日） | 回测区间**第一天** |
| 回测区间不是数据第一天 | 回测区间起点日**的前一交易日** | 回测区间**第一天** |

### 设计原理

1. **确保第一天就有收益率**：区间第一天就有参照物（基准日资产）
2. **收益率计算连续性**：`收益率 = (当日资产 - 基准日资产) / 基准日资产`
3. **与行业惯例一致**：类似同花顺、聚源等平台的做法

## 3. 回测回调函数

### `on_backtest_finished(context, indicator)`

回测完成后的回调函数，通过 `indicator` 参数获取绩效指标。

```python
def on_backtest_finished(context, indicator):
    print('*'*50)
    print('回测已完成')
    print(f"累计收益率: {indicator.get('pnl_ratio')}")
    print(f"年化收益率: {indicator.get('pnl_ratio_annual')}")
    print(f"夏普比率: {indicator.get('sharp_ratio')}")
    print(f"最大回撤: {indicator.get('max_drawdown')}")
```

## 4. 绩效指标 (Indicator)

gm包通过protobuf定义绩效指标，数据结构如下：

### 主指标 (Indicator)

定义位置：`gm/pb/performance_pb2.pyi`

| 字段 | 类型 | 说明 |
|------|------|------|
| `pnl_ratio` | float | 累计收益率 (pnl/cum_inout) |
| `pnl_ratio_annual` | float | 年化收益率 |
| `sharp_ratio` | float | 夏普比率 |
| `max_drawdown` | float | 最大回撤 |
| `risk_ratio` | float | 风险比率（年化波动率） |
| `open_count` | int | 开仓次数 |
| `close_count` | int | 平仓次数 |
| `win_count` | int | 盈利次数 |
| `lose_count` | int | 亏损次数 |
| `win_ratio` | float | 胜率 |
| `calmar_ratio` | float | 卡玛比率 |
| `account_id` | str | 账号ID |

### 周期指标 (IndicatorDuration)

定义位置：`gm/pb/performance_pb2.pyi`

| 字段 | 类型 | 说明 |
|------|------|------|
| `pnl_ratio` | float | 周期收益率快照 |
| `pnl` | float | 收益快照 |
| `fpnl` | float | 浮盈浮亏快照 |
| `cash` | float | 资金快照 |
| `nav` | float | 净值快照 |
| `cum_pnl` | float | 周期累计盈亏 |
| `cum_buy` | float | 周期累计买入额 |
| `cum_sell` | float | 周期累计卖出额 |
| `cum_commission` | float | 周期累计手续费 |

## 5. 数据获取函数

### `history()` - 查询历史行情

```python
history(
    symbol,              # 标的代码，如 'SHSE.000300'
    frequency,           # 频率：'1d', '1m', '5m', '60m' 等
    start_time,          # 开始时间
    end_time,           # 结束时间
    fields=None,        # 字段：'open,high,low,close,eob'
    skip_suspended=True, # 是否跳过停牌
    fill_missing=None,   # 填充缺失值
    adjust=None,         # 复权模式
    adjust_end_time='', # 复权截止时间
    df=False            # 是否返回DataFrame
)
```

### `history_n()` - 获取最近N条数据

```python
history_n(
    symbol,           # 标的代码
    frequency,        # 频率
    count,            # 数据条数
    end_time=None,    # 截止时间
    fields=None,      # 字段
    adjust=None,      # 复权模式
    adjust_end_time='' # 复权截止时间
)
```

### 示例

```python
# 获取沪深300历史数据
df = history(
    symbol='SHSE.000300',
    frequency='1d',
    start_time='2019-01-01',
    end_time='2024-12-31',
    fields='open,high,low,close,eob',
    adjust=ADJUST_PREV,
    df=True
)

# 获取回测中最近N条数据
new_price = history_n(
    symbol='SHSE.000300',
    frequency='1d',
    count=1,
    end_time=context.now,
    adjust=ADJUST_PREV,
    adjust_end_time=context.backtest_end_time,
    df=False
)[0]['open']
```

## 9. 与ft2框架对比

ft2框架的基准日逻辑与gm包**完全一致**：

| 场景 | ft2框架 | gm包 |
|------|---------|------|
| 全部数据 | `benchmark_date = date[0]` | 基准日 = 区间起点日 |
| 自定义区间 | `benchmark_date = 区间起点前一交易日` | 基准日 = 区间起点前一交易日 |

ft2框架实现位置：`core/analyzer.py` - `_slice_data_by_range()` 方法

## 10. 注意事项

1. **gm包是商业闭源库**：核心计算逻辑在C++ SDK层实现
2. **token需要申请**：需要从掘金量化官网申请token才能使用
3. **数据依赖终端**：回测数据需要预先下载到本地终端
4. **撮合模式选择**：
   - `match_mode=0`（开盘价）更保守，模拟开盘买入的情况
   - `match_mode=1`（收盘价）更激进，模拟收盘价成交
