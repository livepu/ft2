# 回测框架使用指南

## 1. 框架概述

这是一个基于Python的量化交易回测框架，设计上兼容掘金API，支持多频率数据和多种策略类型。框架采用模块化设计，核心组件包括上下文管理、账户管理、回测引擎和结果分析器。

### 1.1 主要特点

- **兼容掘金API**：便于策略迁移和实盘对接
- **模块化设计**：各组件独立，便于扩展和修改
- **支持多频率数据**：支持1m、5m、15m、30m、60m、1d等多种数据频率
- **强大的分析功能**：自动计算各种回测指标，生成可视化HTML报告
- **灵活的数据管理**：支持多种数据格式和数据源

## 3. 核心组件介绍

### 3.1 Context（上下文管理）

**功能**：管理全局状态、数据缓存和订阅信息

**主要方法**：

- `subscribe(symbols, freq='1d', count=100, fields=None, format='df')`：订阅数据
- `unsubscribe(symbols, freq='1d')`：取消订阅
- `data(symbol, frequency, count=1, fields=None)`：获取历史数据

**使用示例**：

```python
from ft2 import context

# 订阅数据
context.subscribe(['SHSE.000300', 'SZSE.399006'], freq='1d', count=200)

# 获取历史数据
df = context.data('SHSE.000300', '1d', count=100, fields=['open', 'high', 'low', 'close'])
```

### 3.2 AccountManager（账户管理）

**功能**：处理订单、管理持仓和记录交易

**主要方法**：

- `order_volume(symbol, volume, price=None)`：按数量下单
- `order_percent(symbol, percent, price=None)`：按总资产比例下单
- `get_account()`：获取账户信息
- `get_position(symbol=None)`：获取持仓信息
- `get_orders(start_query_time=None, end_query_time=None)`：获取历史订单

**使用示例**：

```python
from ft2 import account

# 按数量下单
account.order_volume('SHSE.000300', 100, 3000.0)

# 按比例下单（买入总资产的10%）
account.order_percent('SHSE.000300', 0.1)

# 获取账户信息
account_info = account.get_account()
print(f"总资产：{account_info['nav']}")
print(f"可用资金：{account_info['cash']}")

# 获取持仓信息
position = account.get_position('SHSE.000300')
print(f"持仓数量：{position['volume']}")
print(f"持仓成本：{position['cost_price']}")
```

### 3.3 Engine（回测引擎）

**功能**：加载历史数据并按时间轴运行策略

**主要方法**：

- `add_data(symbol, freq, data)`：添加历史数据
- `set_cache_count(cache_count)`：设置缓存长度
- `run(strategy_class, start_time, end_time)`：运行回测

**使用示例**：

```python
from ft2 import engine

# 添加历史数据
engine.add_data('SHSE.000300', '1d', df)

# 运行回测
engine.run(MyStrategy, start_time=start_time, end_time=end_time)
```

### 3.4 AccountAnalyzer（结果分析）

**功能**：分析回测结果，生成各种指标和HTML报告

**主要方法**：

- `calculate_return_rate(time_interval=None)`：计算收益率
- `calculate_annualized_return(time_interval=None)`：计算年化收益率
- `calculate_sharpe_ratio(risk_free_rate=0.02, time_interval=None)`：计算夏普比率
- `calculate_max_drawdown()`：计算最大回撤
- `to_html_report(report_name="回测报告", output_dir=".")`：生成HTML报告

**使用示例**：

```python
from ft2 import AccountAnalyzer

# 创建分析器实例
analyzer = AccountAnalyzer(account)

# 计算回测指标
return_rate = analyzer.calculate_return_rate()
annualized_return = analyzer.calculate_annualized_return()
sharpe_ratio = analyzer.calculate_sharpe_ratio()
max_drawdown = analyzer.calculate_max_drawdown()

# 生成HTML报告
analyzer.to_html_report(report_name="动量策略回测报告")
```

## 4. 快速开始

### 4.1 创建策略类

```python
from ft2 import context, account, engine
import pandas as pd

class 动量策略:
    def __init__(self):
        # 设置回测模式
        context.mode = 'backtest'
        
        # 订阅数据
        self.symbols = ['SHSE.000300', 'SZSE.399006']
        context.subscribe(self.symbols, freq='1d', count=200)
        
        # 加载历史数据（此处使用模拟数据，实际使用时应替换为真实数据源）
        for symbol in self.symbols:
            # 生成模拟数据
            dates = pd.date_range('2020-01-01', '2024-12-31', freq='D')
            closes = pd.Series(range(len(dates)), index=dates)
            df = pd.DataFrame({'close': closes, 'eob': dates})
            
            # 添加数据到引擎
            engine.add_data(symbol, '1d', df)
    
    def on_bar(self, context, bars):
        # 获取最新数据
        for symbol in self.symbols:
            df = context.data(symbol, '1d', count=10)
            
            # 简单的动量策略：如果价格上涨，则买入
            if len(df) >= 2:
                if df['close'].iloc[-1] > df['close'].iloc[-2]:
                    account.order_volume(symbol, 100, df['close'].iloc[-1])
                else:
                    # 如果价格下跌，则卖出所有持仓
                    position = account.get_position(symbol)
                    if position['volume'] > 0:
                        account.order_volume(symbol, -position['volume'], df['close'].iloc[-1])
```

### 4.2 运行回测

```python
import datetime

```

## 6. 回测报告解读

### 6.1 报告结构

回测报告包含以下主要部分：

1. **回测概览**：显示报告名称、回测时间范围等基本信息
2. **业绩指标**：包括累计收益率、年化收益率、夏普比率、最大回撤等
3. **净值曲线**：展示策略净值的变化趋势
4. **交易记录**：详细的成交记录，包括日期、标的、价格、数量等
5. **盈亏分析**：显示最大盈利和最大亏损的交易

### 6.2 关键指标解释

- **累计收益率**：策略期间的总收益率
- **年化收益率**：将累计收益率年化后的结果
- **夏普比率**：(年化收益率 - 无风险利率) / 年化波动率，衡量风险调整后的收益
- **最大回撤**：策略净值从峰值到谷值的最大跌幅
- **盈亏比**：平均盈利与平均亏损的比值

## 8. 性能优化建议

1. **减少数据缓存长度**：对于长期回测，可以适当减少数据缓存长度
2. **优化策略逻辑**：避免在`on_bar`方法中执行复杂的计算
3. **使用适当的数据频率**：根据策略需求选择合适的数据频率，避免使用过高频率的数据
4. **批量加载数据**：一次性加载所有需要的数据，避免频繁的数据访问

## 9. 实盘对接

框架设计上兼容掘金API，因此策略可以相对容易地迁移到实盘环境。需要注意以下几点：

1. 确保策略使用的API在实盘中可用
2. 调整交易参数以适应实盘环境
3. 添加适当的风险控制机制
4. 考虑实盘延迟和滑点的影响
